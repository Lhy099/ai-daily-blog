## TL;DR
本文深度解析深度学习中两大核心架构：CNN（卷积神经网络）和RNN（循环神经网络）。CNN是AI的"眼睛"，通过局部特征提取处理图像数据；RNN专为序列数据设计，通过LSTM和GRU变体解决长期依赖问题。两者在现代大模型中各有应用场景。

## 一、CNN（卷积神经网络）：AI的"眼睛"

### 1.1 核心思想

卷积神经网络（Convolutional Neural Network, CNN）是让计算机拥有"视觉"的核心技术，最擅长处理像图像这样的网格数据。

**工作原理：**
想象手里拿着一个小方框（称为"卷积核"或"过滤器"），在一张大照片上滑动。每滑动到一个位置，就只关注这个方框里的内容，寻找特定的特征，比如边缘、线条或颜色斑点。

### 1.2 架构组成

| 层级 | 功能 | 说明 |
|------|------|------|
| **卷积层** | 特征提取 | 应用滤波器提取边缘、纹理等局部特征 |
| **池化层** | 降维压缩 | 减少特征图空间维度，保留主要信息 |
| **全连接层** | 分类决策 | 将提取的特征映射到输出类别 |
| **激活函数** | 非线性变换 | 通常使用ReLU、tanh等 |

### 1.3 核心特点

**1. 局部感受野**
- 卷积核只关注局部区域（如3×3像素）
- 通过滑动窗口扫描整张图像

**2. 权重共享**
- 同一个卷积核在整个图像上共享参数
- 大幅减少参数量，提高训练效率

**3. 平移不变性**
- 对目标位置变化不敏感
- 无论猫在图片哪个位置都能识别

**4. 层次化特征提取**
- 浅层：检测边缘、线条等简单特征
- 中层：组合成形状（圆圈、方形）
- 深层：识别复杂物体（猫的耳朵、汽车的轮子）

### 1.4 数学原理

卷积操作的核心公式：

```
output(i, j, k) = bias(k) + Σ(input(i+p, j+q) × weight(p, q, k))
```

其中：
- `output(i, j, k)`：输出特征图在位置(i,j)的第k个通道值
- `input(i+p, j+q)`：输入数据的局部区域
- `weight(p, q, k)`：卷积核权重
- `bias(k)`：偏置项

### 1.5 代表模型

| 模型 | 年份 | 特点 |
|------|------|------|
| **LeNet** | 1998 | 最早的CNN架构，5层结构 |
| **AlexNet** | 2012 | 8层，ReLU+Dropout，开启深度学习热潮 |
| **VGGNet** | 2014 | 16-19层，小卷积核(3×3)堆叠 |
| **ResNet** | 2015 | 残差连接，可训练152+层 |
| **EfficientNet** | 2019 | 复合缩放，效率与精度平衡 |

### 1.6 应用场景

- **图像分类**：识别图片中的物体类别
- **目标检测**：定位并识别图像中的多个对象
- **图像分割**：像素级别的分类
- **人脸识别**：安防、身份验证
- **医学影像**：X光、CT、MRI分析

## 二、RNN（循环神经网络）：序列数据的专家

### 2.1 核心思想

循环神经网络（Recurrent Neural Network, RNN）专为序列数据设计，通过**隐藏状态**传递历史信息，能够处理变长输入，捕捉时间依赖关系。

**核心公式：**
```
h_t = tanh(W·[x_t, h_{t-1}] + b)
```

其中：
- `h_t`：当前时刻隐藏状态
- `x_t`：当前时刻输入
- `h_{t-1}`：上一时刻隐藏状态
- `W`：权重矩阵，**参数共享**

### 2.2 RNN的致命缺陷：梯度消失

标准RNN在处理长序列时存在**梯度消失或爆炸问题**：

当序列较长时，梯度在反向传播中会指数级衰减：
```
∂L/∂h_{t-k} ≈ 0  (当k很大时)
```

这导致模型难以学习长期依赖关系——例如理解长篇文章中开头和结尾的关联。

### 2.3 LSTM：长短期记忆网络

为解决梯度消失问题，**LSTM（Long Short-Term Memory）**于1997年提出，通过门控机制控制信息流。

**核心组件：**

| 门控 | 功能 | 公式 |
|------|------|------|
| **遗忘门** | 决定丢弃哪些旧信息 | f_t = σ(W_f·[h_{t-1}, x_t] + b_f) |
| **输入门** | 决定存储哪些新信息 | i_t = σ(W_i·[h_{t-1}, x_t] + b_i) |
| **输出门** | 决定输出哪些信息 | o_t = σ(W_o·[h_{t-1}, x_t] + b_o) |
| **细胞状态** | 长期记忆的"传送带" | C_t = f_t ⊙ C_{t-1} + i_t ⊙ Ĉ_t |

**关键创新：**
- **细胞状态（Cell State）**：像传送带一样贯穿整个链式系统
- **门控机制**：通过sigmoid函数输出0-1之间的值，控制信息流动比例
- **三个输入**：x_t（当前输入）、h_{t-1}（上一输出）、C_{t-1}（上一记忆）
- **两个输出**：h_t（当前输出）、C_t（当前记忆）

### 2.4 GRU：门控循环单元

**GRU（Gated Recurrent Unit）**于2014年提出，是LSTM的简化版本。

**与LSTM的区别：**

| 特性 | LSTM | GRU |
|------|------|-----|
| **门控数量** | 3个（遗忘、输入、输出） | 2个（更新、重置） |
| **状态类型** | 隐藏状态+细胞状态 | 仅隐藏状态 |
| **参数量** | 较多 | 约为LSTM的2/3 |
| **训练速度** | 较慢 | 更快 |
| **性能** | 长序列更优 | 短序列相当 |

**GRU的核心公式：**

```
更新门：z_t = σ(W_z·[h_{t-1}, x_t])
重置门：r_t = σ(W_r·[h_{t-1}, x_t])
候选状态：h̃_t = tanh(W·[r_t ⊙ h_{t-1}, x_t])
最终状态：h_t = (1-z_t)⊙h_{t-1} + z_t⊙h̃_t
```

### 2.5 三种RNN变体对比

| 模型 | 参数量 | 计算复杂度 | 长期依赖能力 | 适用场景 |
|------|--------|-----------|-------------|----------|
| **标准RNN** | 最少 | 低 | 弱（<10步） | 简单序列 |
| **LSTM** | 多 | 高 | 强（100+步） | 长文本、语音 |
| **GRU** | 中等 | 中等 | 较强（50+步） | 平衡效率与性能 |

### 2.6 应用场景

- **自然语言处理**：机器翻译、文本生成、情感分析
- **语音识别**：将语音转换为文字
- **时间序列预测**：股价、天气、流量预测
- **音乐生成**：创作旋律和和弦进行
- **视频分析**：动作识别、行为预测

## 三、CNN vs RNN：核心对比

### 3.1 数据处理方式的差异

| 维度 | CNN | RNN |
|------|-----|-----|
| **数据类型** | 网格数据（图像） | 序列数据（文本、时间序列） |
| **处理逻辑** | 局部感受野，空间特征提取 | 时序依赖，状态传递 |
| **参数共享** | 空间维度共享卷积核 | 时间维度共享权重矩阵 |
| **并行能力** | 高（可GPU加速） | 低（必须时序执行） |

### 3.2 特征提取能力

**CNN的优势：**
- 局部特征提取能力强
- 平移不变性
- 可并行计算，训练效率高
- 层次化特征表示

**CNN的局限：**
- 难以建模长距离依赖
- 对序列顺序不敏感

**RNN的优势：**
- 天然适合序列数据
- 可变长输入/输出
- 能捕捉时序依赖关系

**RNN的局限：**
- 难以并行（必须顺序计算）
- 长期依赖问题（LSTM/GRU缓解但未完全解决）
- 训练慢，难以扩展到深层

### 3.3 现代大模型中的角色

**Transformer的崛起：**

现代大模型（如GPT、BERT）主要基于**Transformer架构**，它结合了CNN和RNN的优点：

| 特性 | Transformer |
|------|-------------|
| **核心机制** | 自注意力（Self-Attention） |
| **依赖建模** | 全局依赖，任意距离 |
| **并行能力** | 完全可并行 |
| **计算复杂度** | O(n²) 注意力计算 |

**但CNN和RNN仍有其价值：**
- **CNN**：在计算机视觉中仍是主力（ResNet、EfficientNet）
- **RNN**：轻量级序列任务、边缘设备
- **混合架构**：CNN-Transformer（视觉）、RNN-Transformer（语音）

## 四、架构选择指南

### 4.1 按数据类型选择

| 数据类型 | 推荐架构 |
|----------|----------|
| 图像 | CNN、Vision Transformer |
| 文本/语音 | Transformer、LSTM、GRU |
| 时间序列 | LSTM、GRU、Transformer |
| 视频 | 3D-CNN、CNN+RNN混合 |
| 图结构 | GNN（图神经网络） |

### 4.2 按任务需求选择

| 任务需求 | 推荐方案 |
|----------|----------|
| 需要实时推理 | CNN、GRU |
| 需要长距离依赖 | Transformer、LSTM |
| 资源受限（移动端） | MobileNet、GRU |
| 最高精度 | Transformer、ResNet |

## 五、总结与展望

### 5.1 核心结论

1. **CNN**是计算机视觉的基石，通过局部卷积和权重共享高效提取空间特征
2. **RNN家族**（RNN/LSTM/GRU）是序列建模的先驱，通过门控机制解决长期依赖
3. **Transformer**正在统一NLP和CV，但CNN和RNN在特定场景仍有不可替代的价值

### 5.2 发展趋势

- **效率优化**：轻量化CNN（MobileNet）、高效Transformer（FlashAttention）
- **混合架构**：CNN-Transformer、RNN-Transformer结合各自优势
- **硬件协同**：针对特定架构设计AI芯片（如NVIDIA的Tensor Core优化Transformer）
